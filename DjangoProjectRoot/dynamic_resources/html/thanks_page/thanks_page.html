Why I choose Django+Python?
  • Simple, powerful and effective.
  • Python is a interpreted programming language. It is easy to learn, use and change.
  • Although Python is dynamic typed language, it still has typing system. USE IT! Not IGNORE IT!
  • Separate Front and End seems to be a joke when doing REAL Engineering.
    • Because a lot of data paraphrasing and transformation work needs to be done on the server side. Dynamic HTML generate and translate work need to be done also by the server side.
    • JavaScript does not support typing system. Although there is a typing system in TypeScript, it is not as mature and flexible as Python's typing system. And the most important thing is that: TypeScript is a compiled language. It still need to transpile to JavaScript. The browser does not permit TypeScript running directly.
    • Packing and Depacking things is doing the same thing twice. If there is a direct, specific, one-only communication channel. This is not ARP or IP/TCP, we do not need to add tags and organize data in extra layers. This also slows down the communication. It is like you know English and Chinese and communicate with a foreigner who only knows English. But you guys communicate through the third language like French.
    • Use template is already enough for module engineering.
Why not Java/SpringBoot, Php or Ruby/Rails?
  • Java is too much, and not truely flexible and opensource, which gives a bad community. Just like 5070 TI Graphic Card does not support Linux adjust the screen brightness in HP MAX. It is the driver problem! (Oracle and Nvidia F**k you!). Also, Java's packageing system is like C, which is very chaos.
  • Php does not work well with other languages in some cases I think.
  • Ruby does not have much package support.
  • But I do like programming languages cooperate with each other. Like C for some flexible system, Python for quick development and code readability. In my Undergraduate final year Project, I do use Python for AI module, Rust for rough data cleaning (almost 1TB data, python is too slow), Shell for quick command serialization, and R for final data visualization and analysis. They work pretty well together. And another reason is library is not universal. Like Python does not have a very good visualization library, but R does.

Information are blocked in some cases. So share the information inside the cage is better than people breaks the cages and get the information from the outside.

  • Another important thing is that, maintain a complicated system costs a lot of time. I do not think it is worthy enough. This website is just a simple introduction of me. It is more like a map/tourist guide/Dish menu. When you want to see real pictures, go to the real place or order the food to have a try.

  • Finally, maintain a big system is not easy, especially with database.
